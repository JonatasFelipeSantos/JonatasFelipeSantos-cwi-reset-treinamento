O QUE SÃO E PARA QUE SERVEM OS TESTES DE UNIDADES ? 

TESTE DE UNIDADE É UM PROCESSO DE DESENVOLVIMENTO DE SOFTWARE EM QUE AS MENORES PARTES TESTÁVEIS DE UMA APLICAÇÃO, CHAMADAS UNIDADES, SÃO EXAMINADAS INDIVIDUAL E INDEPENDENTEMENTE PARA UMA OPERAÇÃO. 

AO UTILIZAR ESTE PROCESSO É POSSIVEL OBTER TESTES CONTINUOS E REVISÃO DO QUE ACABOU DE SER ALTERADO NA APLICAÇÃO, EVITANDO QUEBRAR FUNCIONALIDADES QUE JÁ ESTAVAM FUNCIONANDO. ALÉM DISSO, SÃO NORMALMENTE RÁPIDOS E FÁCEIS DE DEPURAR. 

PRÓS E CONTRAS DOS TESTES DE UNIDADE

PRÓS 
	* SÃO EXECUTADOS EM S. OU MS. 
	* PROVÊM FEEDBACK IMEDIATO 
	* ISOLAMENTO DE FALHAS / FÁCIL DE DEPURAR
	* SÃO CONFIÁVEIS 
	* GUIAM O DESENVOLVIMENTO 
	* AJUDAM NA REFATORAÇÃO 
	* PODEM EXISTIR TANTO NO BACK-END QUANTO NO FRONT-END

CONTRAS

	* NÃO SIMULAM O USO DA APLICAÇÃO DE PONTA-A-PONTA

TDD(DESENVOLVIMENTO GUIADO POR TESTES): TDD É UM PROCESSO DE DESENVOLVIMENTO DE SOFTWARE CRIADO POR KENT BECK, QUE DEPENDE DA REPETIÇÃO DE UM PEQUENO CICLO DE DESENVOLVIMENTO: REQUISITOS SÃO TRANSFORMADOS EM CASOS DE TESTES ESPECÍFICOS E O SOFTWARE É MELHORADO PARA FAZER O NOVO TESTES PASSAR. 


REFATORAÇÃO: É UM PROCESSO DE REESTRUTURAÇÃO DE CÓDIGO QUE MUDA SUA ESTRUTURA SEM MUDAR O COMPORTAMENTO. 

A REFATORAÇÃO PERMITE A MELHORIA DE ATRIBUTOS NÃO FUNCIONAIS DO SOFTWARE, TORNANDO-O MAIS LEGÍVEL E MENOS COMPLEXO. 

É UM PROCESSO POSSÍVEL DEVIDO A ESCRITA DE TESTES DE UNIDADE, SEJA UTILIZANDO OU NÃO TDD, APESAR DO TDD SER ENCORAJADO. 


TESTES DE UNIDADE NO PROCESSO DE CI

* BACK-END 
* FRONT-END 
* SÃO EXECUTADOS NOS PRIMEIROS ESTÁGIOS DO PIPELINE 

COBERTURA DE CÓDIGO: É UMA MEDIDA UTILIZADA PARA DESCREVER O GRAU EM QUAL O CÓDIGO FONTE DE UM PROGRAMA É EXAMINADO POR TESTES. 

UM PROGRAMA COM ALTA COBERTURA DE CÓDIGO TEVE MAIS DE SEU CÓDIGO FONTE EXECUTADO POR TESTES, O QUE FAZ COM QUE ESTE TENHA MENOS CHANCES DE POSSUIR ERROS NÃO DETECTADOS, QUANDO COMPARADOS COM UM PROGRAMA COM BAIXA COBERTURA.
